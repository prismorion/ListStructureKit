Библиотека классов ListStructureKit (LSK) для работы со списочными структурами.

Классы SinglyLinkedListLSK<T> и DoublyLinkedListLSK<T> представляет однонаправленный и двунаправленный списки соответственно. Они содержат поля First и Last для указания на первый и последний элементы списка соответственно, а также поле Size для отслеживания размера списка. Классы содержат конструкторы для инициализации списка элементами из массива или коллекции. Методы AddFirst и AddLast добавляют элементы в начало и конец списка соответственно. Методы AddBefore и AddAfter добавляют новые элементы перед или после указанного элемента. Методы RemoveFirst и RemoveLast удаляют и возвращают элементы из начала и конца списка, а метод Remove удаляет первое вхождение указанного элемента. Метод Clear очищает список. Метод Concat объединяет два списка в один. Методы Serialization и Deserialization сериализуют и десериализуют список в/из файл JSON соответственно. Класс также реализует интерфейсы IEnumerable<T> и IEnumerable для перечисления элементов списка. Данные классы хоть и имеют одинаковые методы, но их внутренняя архитектура различается и является открытой (но по-прежнему реализует свойство ООП – инкапсуляцию, что позволяет взаимодействовать с данными и не нарушать их целостность), что позволяет использовать каждый из них для решения различных задач.

Класс StackLSK<T> представляет стек. Он содержит приватное поле Top для указания на верхний элемент стека и публичное свойство Size для отслеживания размера стека. Класс имеет конструкторы для инициализации стека элементами из массива или коллекции. Метод Push добавляет элемент в вершину стека, а метод Pop удаляет и возвращает элемент из вершины стека. Метод Peek возвращает элемент из вершины стека без его удаления. Метод Clear очищает стек. Метод IsEmpty проверяет стек на пустоту. Методы Serialization и Deserialization сериализуют и десериализуют стек в/из файл JSON соответственно. Класс также реализует интерфейсы IEnumerable<T> и IEnumerable для перечисления элементов стека.

Класс QueueLSK<T> представляет очередь и содержит поля First и Last для указания на первый и последний элементы очереди соответственно, а также свойство Size для отслеживания размера. Он имеет конструкторы для инициализации очереди элементами из массива или коллекции. Метод Enqueue добавляет элемент в конец очереди, а метод Dequeue удаляет и возвращает элемент из начала очереди. Метод Peek возвращает элемент из начала очереди без его удаления. Метод Clear очищает очередь. Метод Concat объединяет две очереди в одну. Методы Serialization и Deserialization сериализуют и десериализуют очередь в/из файл JSON соответственно. Класс также реализует интерфейсы IEnumerable<T> и IEnumerable для перечисления элементов очереди.

Класс DequeLSK<T> представляет дек (двустороннюю очередь) и содержит свойства First и Last для указания на первый и последний элементы дека соответственно, а также свойство Size для отслеживания размера. Он имеет конструкторы для инициализации дека элементами из массива или коллекции. Методы PushFirst и PushLast добавляют элемент в начало и конец дека соответственно. Методы PopFirst и PopLast удаляют и возвращают элемент из начала и конца дека соответственно. Методы PeekFirst и PeekLast возвращают элемент из начала и конца дека без их удаления. Метод Clear очищает дек. Методы Serialization и Deserialization сериализуют и десериализуют очередь в/из файл JSON соответственно. Класс также реализует интерфейсы IEnumerable<T> и IEnumerable для перечисления элементов дека.

Класс DrumLSK<T> представляет барабан, который хранит элементы в виде кольцевой структуры. Он содержит приватное поле Top для указания на вершину барабана и свойство Capacity для хранения емкости барабана. Конструктор принимает емкость барабана и инициализирует его. Метод InitializeDrum создает кольцевую структуру барабана, соединяя каждый элемент с предыдущим и следующим. Методы RotateClockwise и RotateCounterClockwise вращают барабан по часовой и против часовой стрелки соответственно. Метод Write записывает значение в текущий элемент (вершину) барабана, а метод Read возвращает значение текущего элемента. Метод Clear очищает барабан, устанавливая значение каждого элемента в значение по умолчанию для типа T. Метод Contains проверяет наличие указанного элемента в барабане и в случае наличия такового проворачивает барабан на данный элемент, тем самым устанавливая его в Top. Методы Serialization и Deserialization сериализуют и десериализуют очередь в/из файл JSON соответственно.
